AI 时代，增长是每个人的基本能力

先说个反共识的判断，不吹不黑：

在这个 AI 高速发展的时代，代码这个杠杆，越来越不值钱了。

为什么？AI Coding 让软件的边际成本越来越小，小到几乎不存在。以前一个功能要憋一周，现在一句 prompt 就出来了。当所有人都能快速造东西，会写代码本身就不再是护城河。

那护城河还有什么？是增长

  • 产品即增长

  • 增长即用户

一、越是红利低迷期，越是增长的时刻

现在市场低迷，AI 有点泡沫。但大家记住：越是市场低迷期的时候，越是增长被重视的时刻

红利好的年代，躺着都涨，没人在乎增长怎么做。可现在不一样了：人口红利见顶，流量越来越贵，融资越来越难。蛋糕不变大了，你想多吃一口，就得比别人更会抢

这时候，增长能力就是真特么的硬通货

二、增长不是某个岗位的事

很多人有个误区，觉得增长是增长团队的活，跟自己没关系。错。

数据驱动产品，产品跨界运营，大家去了解 UI UX，关注商业价值和用户，更牛的是上升到人性的需求洞察……尤其这个时代，每个岗位不再是各扫门前雪了

增长不仅仅是具体某个职能岗位的事情，它越来越成为各个人都应具备的一种底层基本能力

不管你是做产品、写代码、跑运营还是搞设计，最终都得回到一件事上：

围绕产品、围绕用户做增长

离开了增长，再漂亮的功能也是自嗨

三、增长很科学，核心是能落地

3.1 增长科学理论

别把增长想得玄乎。它一点都不玄，反而很科学，就像做实验一样。

而做增长最重要的，就一个词：能落地 & 迭代快跑

说得再天花乱坠，跑不出结果都是白搭

每个时代的增长手段，无非分两类：

  • 新的增长方式：新机会、新洼地。比如 ChatGPT Ads 广告，比如 Agent Skills 从今年 2 月初冒出来的这波红利。这些就是当下的洼地，谁先下手谁先吃到

  • 传统的增长手段：SEO、投流、红人……这些老办法没过时，只是要看你怎么结合当下用。

但不管新的旧的，每个增长手段，都得结合产品阶段和 ICP 用户画像来验证。

别看到别人做啥火就抄啥，先搞清楚自己的产品是什么、付费用户是谁。

3.2 增长实操

大家很多卡在新产品 0 到 1

我很喜欢在 0 到 1 设计产品的时候考虑增长点。比如我要做一个产品的时候，会初步 Reserach 一些关键结论：

  • 能不能做 SEO，SEO 对应的词簇有没有红利 & 规划

  • 一期产品主打的是？够小、够直接不？有人 Google Search 不？Trends 稳不稳定？

  • 等等

所以我说 增长即产品、产品即用户，在做的时候就得考虑，尽量找交集！

四、一套可复用的增长闭环

具体怎么落地？我自己跑下来，就是这么一个闭环：

洞察 → 建立实验 MVP → 分析 → 总结 → 决定是否放大。

先洞察机会在哪，再搭一个最小可行的实验去验证，跑出数据后分析、总结，最后判断这条路值不值得加大投入。

每个手段都这么过一遍，跑通了就放大，跑不通就换下一个。

这就是增长的科学之处：不靠拍脑袋，靠实验。

小结：记住这几条

  • 代码杠杆变小：AI Coding 让软件边际成本趋近于零，会写代码不再是护城河。

  • 增长是硬通货：红利越萎缩，增长越值钱。

  • 增长是底层能力：每个角色都得围绕产品和用户做增长。

  • 核心是能落地：增长很科学，像做实验，跑不出结果都是空谈。

  • 抓新洼地：ChatGPT Ads、Agent Skills 这类新机会，谁先下手谁先吃到。

  • 走通闭环：洞察 → MVP → 分析 → 总结 → 放大，每个手段都这么打一遍。
